## Requirements

```r
install.packages(c("meta", "metafor", "ggplot2", "dplyr", "readr",
                   "gridExtra", "scales", "tibble", "patchwork",
                   "tidyr", "writexl", "knitr", "pacman"))
```

## How to Run

### Step 1 — Primary meta-analysis
```r
setwd("path/to/qml_dermatology")
source("meta_analysis_main.R")
```
Produces:
- Console output: pooled effect, subgroup table, Egger's test result, LOO summary
- `outputs/figures/` — 5 publication-ready figures
- `outputs/tables/Table_S1_subgroup_results.csv`
- `outputs/tables/Table_S2_leave_one_out.csv`
- `outputs/tables/Table_S3_probast_risk_of_bias.csv`
- `outputs/tables/Table_S4_full_extraction.csv`

### Step 2 — All supplementary tables
```r
source("supplementary_tables.R")
```
Produces Tables S1–S10 in `outputs/tables/`.

## Data Dictionary: `prisma_extraction_database.csv`

| Column | Type | Description |
|--------|------|-------------|
| `study_id` | chr | Unique study identifier (S01–S48) |
| `first_author` | chr | First author surname |
| `year` | int | Publication year |
| `journal` | chr | Journal abbreviation |
| `dataset` | chr | Primary dataset used |
| `dataset_size` | int | Total images in dataset |
| `n_classes` | int | Number of classification classes |
| `task_type` | chr | `binary` or `multiclass` |
| `qml_method` | chr | Quantum ML method used |
| `n_qubits` | int | Number of qubits (NA for annealing) |
| `n_layers` | int | Circuit depth (NA for kernels/annealing) |
| `hardware_type` | chr | `Simulator` or `Real` |
| `hardware_platform` | chr | Specific device or framework |
| `error_mitigation` | chr | Error mitigation method or `None` |
| `classical_baseline` | chr | Classical model compared against |
| `qml_accuracy` | dbl | QML top-1 accuracy (%) |
| `classical_accuracy` | dbl | Classical top-1 accuracy (%) |
| `delta_accuracy` | dbl | `qml_accuracy − classical_accuracy` (pp) |
| `balanced_acc_qml` | dbl | Balanced multi-class accuracy, QML (%) |
| `balanced_acc_classical` | dbl | Balanced multi-class accuracy, classical (%) |
| `auroc_qml` | dbl | Area under ROC curve, QML |
| `auroc_classical` | dbl | Area under ROC curve, classical |
| `train_split` | dbl | Training set proportion |
| `val_split` | dbl | Validation set proportion |
| `test_split` | dbl | Test set proportion |
| `code_available` | chr | `Yes` / `No` |
| `probast_participant` | chr | PROBAST domain 1 risk: `Low`/`Moderate`/`High` |
| `probast_predictor` | chr | PROBAST domain 2 risk |
| `probast_outcome` | chr | PROBAST domain 3 risk |
| `probast_analysis` | chr | PROBAST domain 4 risk |
| `overall_risk` | chr | Overall PROBAST risk |
| `notes` | chr | Reviewer notes on study |

## Key Statistical Results (Reference)

| Analysis | Result |
|----------|--------|
| Pooled ∆ (all, random-effects) | +1.2% (95% CI: −0.8% to +3.2%; p = 0.24) |
| Between-study heterogeneity | I² = 74%, τ² = 8.3, p_Q < 0.001 |
| Publication bias (Egger's test) | p = 0.031 |
| Simulator subgroup ∆ | +2.4% (p = 0.024) |
| Real hardware subgroup ∆ | −2.1% (p = 0.114) |
| Weak baseline subgroup ∆ | +5.8% (p = 0.003) |
| Modern baseline subgroup ∆ | −0.9% (p = 0.307) |
| Simulator-to-hardware accuracy drop | −4.1 ± 1.8% |
